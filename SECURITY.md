# Security Policy

## 支持版本

当前项目处于首次开源准备阶段，安全支持范围如下：

| Version | Supported |
| --- | --- |
| 0.1.x | Yes |
| < 0.1.0 | No |

## 报告安全问题

请不要通过公开 Issue 报告安全漏洞。

请通过以下方式联系维护者：

- 首选：GitHub Security Advisory 私密报告（仓库 `Security` 页）。
- 备用：通过维护者在 GitHub Profile 或项目发布页公开列出的安全联系邮箱私密联系。

报告时请尽量包含：

- 影响范围
- 复现步骤
- 可能的攻击路径
- 受影响版本
- 修复建议

## 配置与密钥

- 不要提交真实访问 Token、Cookie、内部地址或生产 API 配置。
- 示例配置只能使用占位值。
- 演示账号仅用于本地 mock 模式，不代表生产账号策略。

## 依赖安全扫描

- CI 执行 `npm audit --audit-level=high`，高危及以上漏洞会阻断。
- 漏洞例外必须记录影响范围、临时缓解措施、负责人和复查日期，不允许仅通过降低阈值隐藏问题。

## 响应流程

1. 维护者确认报告并评估影响。
2. 在私有分支准备修复。
3. 增加必要测试或验证步骤。
4. 发布安全修复版本。
5. 在 Release Notes 中说明影响范围和升级建议。
