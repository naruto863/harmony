# 前端可观测性边界

Harmony Admin 当前是纯前端项目，不内置服务端健康检查、指标端点或日志采集服务。本文件说明前端侧预留的错误上报和 TraceId 处理边界。

## TraceId

- `apiClient` 会尽量保留外部 API 返回的 `traceId`。
- 用户反馈、错误提示和错误上报可以带上 `traceId`，方便接入方在自己的服务端日志中定位问题。
- 前端不生成业务审计日志的权威记录；真实审计应由接入方的 API 服务完成。

## 前端错误上报

前端预留 `src/services/errorReportingService.ts`，用于统一整理浏览器错误、TraceId、路由路径和 userAgent。默认不开启远程上报，也不自动采集用户输入内容。

后续如接入 Sentry、OpenTelemetry 或自建错误收集 API，应继续遵守：

- 不上报 token、密码、密钥、完整请求体等敏感信息。
- 对邮箱、手机号、身份证号等个人信息做脱敏或哈希。
- 对高频错误做采样，避免噪声和成本失控。
- 在 README 或部署文档中说明新增环境变量。

## 外部监控

如果使用方接入真实 API、网关或静态托管平台，健康检查、Prometheus 指标、日志聚合和告警规则应在对应平台侧配置。本仓库只负责前端构建产物和浏览器侧错误边界。
