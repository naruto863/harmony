# 审计日志保留与归档策略

## 范围

- `sys_audit_log`：后台管理操作、安全事件和文件下载等操作审计。
- `sys_login_log`：登录成功、失败、账号锁定、租户拒绝等认证审计。

## 保留周期

- 默认在线保留 180 天，满足后台查询和近期问题排查。
- 高合规租户可通过部署配置延长到 365 天或更长；不建议缩短到 90 天以下。
- 删除或归档前必须保留 `trace_id`、`tenant_id`、`user_id`、`result`、`created_at` 等关键索引字段，便于离线追踪。

## 归档策略

1. 每月按 `created_at` 分批导出超过在线保留期的日志。
2. 归档文件按租户和月份分区，例如 `audit/tenant-1/2026-01.ndjson`。
3. 归档内容保持脱敏后的 `details`，禁止重新写入明文 token、密码、密钥、手机号和邮箱。
4. 归档完成并校验条数后，再删除在线表中的对应数据。

## 查询策略

- 在线查询优先使用 `tenant_id + created_at`、`tenant_id + result + created_at`、`trace_id` 索引。
- 生产排障优先使用 `trace_id` 关联 API 响应、应用日志和审计记录。
- 失败审计需要保留 `failure_reason`，但不得包含异常堆栈、token 或请求体敏感字段。
