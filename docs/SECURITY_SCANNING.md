# 依赖与安全扫描策略

## 扫描命令

```bash
npm run security:audit
```

等价命令：

```bash
npm audit --audit-level=high
```

## 阻断规则

- `npm audit --audit-level=high` 发现 high 或 critical 漏洞时阻断 CI。
- 中低风险漏洞进入常规修复队列，不在默认 CI 中阻断。
- 新增依赖必须能解释用途、维护状态和替代方案。

## 例外处理

漏洞例外必须在安全评审中确认，并记录：

- 漏洞编号、依赖名称和版本。
- 不可立即升级的原因。
- 已采取的缓解措施。
- 负责人和复查日期。

不允许通过删除扫描步骤、降低阈值或忽略整类依赖来规避问题。需要临时例外时，应只针对具体 CVE 或具体依赖，并在同一变更中补充复查日期。
